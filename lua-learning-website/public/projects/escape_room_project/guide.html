Lesson 1: Hello, Escape Room!

Print the opening scene of your escape room.

New Concepts

  • print("text") — shows one line of text
  • print() with nothing in the parentheses — prints an empty line (good for spacing)
  • print[[ ...several lines... ]] — prints a block of text exactly as you typed it, even with line breaks

Your mission in this lesson: write the opening of your escape room using only print.

Lesson 2: Adding Colors

Use the shell library to color your text and clear the screen.

New Concepts

  • local shell = require("shell") — loads the shell library so you can use its features. "Require" means "I want to use this."
  • shell.clear() — wipes everything off the screen before your game starts
  • shell.foreground(color) — changes the text color for everything that prints after this line
  • shell.reset() — changes the color back to normal
  • Color names: shell.RED, shell.GREEN, shell.YELLOW, shell.BLUE, shell.CYAN, shell.MAGENTA, shell.WHITE
TYPE THIS CODE Step 1: Load the shell library
Where: at the very top of your code (below the comment banner, above all your print lines)
local shell = require("shell")
This line tells Lua "I want to use the shell library, and I'll call it shell." You only need to do this once per file.
Run it. Nothing looks different yet — but make sure the program still works and prints the same text.
TYPE THIS CODE Step 2: Clear the screen
Where: right below the require line
shell.clear()
Now every run starts on a fresh, clean screen.
Run it. The screen should wipe before showing the scene.
TYPE THIS CODE Step 3: Color the title yellow
Where: around your title line — the two shell lines go ABOVE and BELOW the print("=== THE LOCKED ROOM ===")
Add shell.foreground(shell.YELLOW) just above the title line, and shell.reset() just below it. Your title section should look like this:
shell.foreground(shell.YELLOW)
print("=== THE LOCKED ROOM ===")
shell.reset()
The color stays active until you reset(). Always reset when you're done coloring, or the next lines will also come out yellow!
Run it. The title should glow yellow. Everything else stays the normal color.
TYPE THIS CODE Step 4: Color the goal line red
Where: around the print("You need to find a way out!") line
Same idea — put a foreground line above and a reset line below:
shell.foreground(shell.RED)
print("You need to find a way out!")
shell.reset()
Run it. The goal line should now stand out in red.

Lesson 3: Asking the Player

Wait for the player to type something, then respond.

New Concepts

  • io.read() — pauses the program until the player types something and presses Enter, then gives you back whatever they typed
  • local choice = io.read() — stores what the player typed in a variable called choice. A variable is a named box that holds a value.
  • if choice == "look" then ... end — an if statement. It only runs the code inside when what's in choice is exactly the word "look". Note: two equals signs (==) means "is equal to".
TYPE THIS CODE Step 1: Add a prompt
Where: at the bottom of your file, below everything you wrote in Lesson 2
print()
print("Type 'look' and press Enter to look around.")
This tells the player what to do. The first print() is just for spacing.
Run it. You should see the prompt at the bottom of your scene.
TYPE THIS CODE Step 2: Read what the player types
Where: right below the prompt you just added
local choice = io.read()
This line does two things at once:
  1. io.read() waits for the player to type a word and press Enter.
  2. local choice = stores whatever they typed in a variable named choice.
Run it. The program will now wait for you after printing the prompt. Type anything and press Enter. Nothing visible happens yet — but the program now finishes. That's progress!
TYPE THIS CODE Step 3: React when the player types look
Where: right below the local choice = io.read() line
if choice == "look" then
  print("You spot something shiny under the bed!")
end
This is an if statement. Read it out loud: "If choice equals look, then print this." The end marks where the "if" block stops.
The two spaces before print are called indentation. They aren't required, but they make your code easier to read.
Run it. Try typing look — you should see the shiny message. Try typing anything else — nothing happens (yet).

Lesson 4: Multiple Options

Give the player a menu and react to each choice.

New Concepts

  • elseif — lets you check another condition after your if. You can have as many elseif branches as you need.
  • else — runs when none of the other checks matched. Great for handling bad input.
  • Numbered menus: print numbered options like 1. Do this, 2. Do that, and ask the player to type the number.

Structure:

if choice == "1" then
  -- code for option 1
elseif choice == "2" then
  -- code for option 2
elseif choice == "3" then
  -- code for option 3
else
  -- code for any other input
end
TYPE THIS CODE Step 1: Replace the prompt with a menu
Where: replace the single prompt line print("Type 'look' and press Enter to look around.") with the menu below
print("What do you do?")
print("  1. Look under the bed")
print("  2. Open the drawer")
print("  3. Push the door")
print()
Now the player has real choices. Each line tells them a number to type.
Run it. You should see the three options before the program pauses for input.
TYPE THIS CODE Step 2: Replace your if with a branching menu
Where: replace your existing if choice == "look" then ... end block with the one below
if choice == "1" then
  print("You spot something shiny under the bed!")
elseif choice == "2" then
  print("The drawer is empty except for a spider.")
elseif choice == "3" then
  print("The door won't budge. You need something to open it.")
else
  print("That is not a valid choice.")
end
Each elseif checks a different number. The else at the end runs if the player types something that isn't 1, 2, or 3.
Notice we're comparing to "1" (in quotes) and not 1 (no quotes). io.read() always gives back text — even when it's a number — so we compare to text too.
Run it. Try typing 1, then run again and try 2, then 3, then try potato. Each one should give a different message.

The new elseif branch goes before the final else. The order is: if → elseifs → else → end.

Lesson 5: Making Rooms

Turn each location into a function, and let the player walk between rooms.

New Concepts

  • function name() ... end — a function is a named chunk of code. You can run it later by writing name(). Think of a function as a "named section" you can jump to.
  • bedroom() — this calls the function named bedroom — it runs everything inside.
  • A function can call another function (to move the player to a new location).
  • A function can call itself to show its menu again (for when the player types something invalid, or comes back to the same room).

Why do this? Without functions, you can only run your code top-to-bottom one time. With functions, the player can move back and forth between locations, and invalid input can simply re-show the menu.

TYPE THIS CODE Step 1: Wrap the bedroom code in a function
Where: around everything you have after the local shell = require("shell") line
Add function bedroom() just below the require line, then add end at the very bottom of your file. Everything between those two lines is now "the bedroom". It will look like this:
local shell = require("shell")

function bedroom()
  -- ALL your existing code goes here (shell.clear, prints, if/elseif, etc.)
end
Nothing inside the function runs yet — you've only defined the room. The next steps will actually start the game.
TYPE THIS CODE Step 2: Start the game at the bottom
Where: on a brand-new line at the very end of your file, after the end of the bedroom function
bedroom()
This one line starts the game. It calls the bedroom function, which runs everything inside.
Run it. The game should work exactly as before. Good — we haven't changed the behaviour yet, just rearranged it.
TYPE THIS CODE Step 3: Update each branch to loop back and add a door option
Where: replace the entire if / elseif / else block inside bedroom() with this one

Look at what changed in each branch:
  • After the message, we print "Press Enter to continue", then io.read() waits for Enter.
  • Then we call bedroom() again, which re-shows the menu.
  • The "push the door" branch (option 3) now calls hallway() instead — it moves the player to a new room (we'll build that next).
  • The else branch handles invalid input by going back to the menu.

Lesson 6: Tracking Items

Use true/false variables to remember what the player has picked up — and finally escape!

New Concepts

  • local has_key = false — a boolean variable. A boolean only has two values: true or false. We use them to remember things like "does the player have the key?"
  • has_key = true — change the value later, for example when the player picks up the key.
  • if has_key then ... end — checks a boolean directly. When has_key is true, the code inside runs.
  • if not has_key then ... end — the word not flips a boolean. Runs the code inside only when has_key is false.

The plan for this lesson: add a key to pick up in the bedroom, use it to unlock the gate in the hallway, and write a win scene.

Where: just below local shell = require("shell"), above the function bedroom() line
local has_key = false
The game starts with the player having no key. Putting this variable outside any function means every function can see and change it.
Run it. Nothing visible changes, but confirm the game still works.

function hallway()
  shell.clear()

  shell.foreground(shell.CYAN)
  print("=== DUSTY HALLWAY ===")
  shell.reset()

  print()
  print("A long hallway with torches flickering on the walls.")
  print("At the far end is a heavy iron gate.")
  print()
  print("What do you do?")
  print("  1. Go back to the bedroom")
  if has_key then
    print("  2. Unlock the gate with the golden key")
  else
    print("  2. Look at the iron gate")
  end
  print("  3. Touch the torches")
  print()

  local choice = io.read()

  if choice == "1" then
    bedroom()
  elseif choice == "2" then
    if has_key then
      escape_scene()
    else
      print("The gate is locked. You need a key.")
      print("Press Enter to continue.")
      io.read()
      hallway()
    end
  elseif choice == "3" then
    print("The flames are cold! Strange...")
    print("Press Enter to continue.")
    io.read()
    hallway()
  else
    print("Invalid choice! Press Enter to continue.")
    io.read()
    hallway()
  end
end

function escape_scene()
  shell.clear()
  shell.foreground(shell.GREEN)
  print("=================================")
  print("         YOU ESCAPED!")
  print("=================================")
  shell.reset()
  print()
  print("The golden key turns with a satisfying click.")
  print("The gate swings open and sunlight pours in.")
  print()
  print("You are free!")
end
